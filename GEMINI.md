# Project Instructions

## Before Starting the Task

You **must** create the following files before doing any work:

* `Agent.md`
* `Progress.md`
* `StudentLearning.md`
* `AILearning.md`

---

# Agent.md

### Must Do

* Provide a complete overview of the project structure.
* Document every folder and every important file in the codebase.
* Explain the purpose of each file.
* Mention the responsibility of each file/module.
* Include relationships between files whenever necessary.
* Keep the document updated whenever the project changes.

### Should Do

* Make it easy for an AI to quickly locate the correct file for any requested change.
* Avoid requiring the AI to scan the entire codebase.
* Even if the project contains hundreds of files and thousands of lines of code, the AI should first read `Agent.md` to identify which files need modification.
* Use clear headings and organized sections.

---

# Progress.md

### Must Do

* Maintain a step-by-step execution plan.
* Execute only **one step at a time**.
* Stop after completing each step.
* Before stopping, update `Progress.md`.

### Must Record

* Current completed step.
* Remaining steps.
* Overall project progress.
* Summary of work completed.
* AI confidence score for the completed step.
* Overall confidence score for the project.

### Should Do

* Make it easy to resume work from the last completed step.
* Keep progress entries short, clear, and chronological.

---

# StudentLearning.md

### Must Do

After every completed step, write learning notes for the user.

### Include

* What was done.
* Why it was done.
* The concepts used.
* How the implementation works.
* Important technical details.
* Best practices followed.
* Common mistakes to avoid.

### Goal

Help the user deeply understand what is happening in the project instead of only seeing the final result.

---

# AILearning.md

### Must Do

Maintain a learning log for the AI based on user feedback.

### Include

* Mistakes made by the AI.
* Corrections provided by the user.
* New user preferences.
* Lessons learned from each interaction.
* Rules that should be followed in future tasks.

### Should Do

* Update this file whenever the AI makes a mistake or receives new guidance.
* Use previous lessons to avoid repeating the same mistakes.
* Continuously improve future responses based on the accumulated learning.

---

# General Rules

## Must

* Create all four files before starting any implementation.
* Keep all files updated throughout the project.
* Follow the project plan one step at a time.
* Stop after completing each step and update all relevant files.

## Should

* Keep documentation clear, organized, and easy to navigate.
* Make the project easy for both humans and AI to understand.
* Reduce unnecessary code scanning by relying on the documentation.
* Continuously improve documentation as the project evolves.
